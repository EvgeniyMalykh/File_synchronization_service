# Сервис синхронизации файлов

Сервис для синхронизации файлов между локальной папкой и облачным хранилищем Яндекс.Диск.

## Возможности

- Автоматическая загрузка новых файлов в облако
- Перезапись изменённых файлов
- Удаление из облака файлов, удалённых локально
- Периодическая синхронизация с заданным интервалом
- Логирование всех операций

## Установка

```bash
pip install -r requirements.txt
```

## Настройка

Отредактируйте файл `config.ini`:

```ini
[sync]
local_folder = /path/to/local/folder
cloud_folder = /backup
token = YOUR_YANDEX_DISK_OAUTH_TOKEN
sync_period = 60
log_file = sync.log
```

### Параметры

| Параметр | Описание |
|----------|----------|
| `local_folder` | Путь к синхронизируемой папке на локальном компьютере |
| `cloud_folder` | Имя папки в Яндекс.Диске |
| `token` | OAuth-токен доступа к Яндекс.Диску |
| `sync_period` | Период синхронизации в секундах |
| `log_file` | Путь к файлу лога |

### Получение OAuth-токена

1. Перейдите на [oauth.yandex.ru](https://oauth.yandex.ru/)
2. Создайте приложение с правами на Яндекс.Диск
3. Получите OAuth-токен

## Запуск

```bash
python main.py
```

## Структура проекта

```
file-sync-service/
├── main.py             # Главный модуль (точка входа)
├── cloud_storage.py    # Класс для работы с Яндекс.Диском
├── sync.py             # Логика синхронизации
├── config.ini          # Файл конфигурации
├── requirements.txt    # Зависимости
└── README.md           # Документация
```

## Архитектура

- **`cloud_storage.py`** — класс `YandexDiskStorage` с методами `load()`, `reload()`, `delete()`, `get_info()`. Вынесен в отдельный файл для возможности замены на другой сервис.
- **`sync.py`** — функции для сравнения локальных и облачных файлов, определения изменений и выполнения синхронизации.
- **`main.py`** — чтение конфигурации, настройка логирования, валидация параметров, запуск цикла синхронизации.
